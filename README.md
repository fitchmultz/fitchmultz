## Pi extension suite

I maintain small Pi extensions that solve specific agent-workflow problems: keeping long sessions goal-directed, giving agents controlled browser access, handing off difficult work to ChatGPT web sessions, and smoothing session/provider workflows.

The Pi repos are intentionally narrow. They are meant to compose with the local agent harness rather than become large standalone apps.

## Larger systems work

- [`CueLoop`](https://github.com/fitchmultz/cueloop) turns one-off coding-agent requests into local queue items with phases, checks, review, and archiveable state.
- [`AI Control Plane`](https://github.com/fitchmultz/ai-control-plane) is a reference implementation for host-first AI gateway controls, telemetry, detections, and evidence-producing operator workflows.
- [`Spartan Scraper`](https://github.com/fitchmultz/spartan-scraper) is a local-first scraping workbench for scrape, crawl, and research workflows.
- [`Morphio`](https://github.com/fitchmultz/morphio) is an AI-assisted media/content workflow monorepo with FastAPI, Next.js, reusable adapters, and native routines.
- [`Splunk TUI`](https://github.com/fitchmultz/splunk-tui) is a Rust CLI/TUI for Splunk operations, diagnostics, and local-first verification.

## How I try to make projects reviewable

- Show the outcome before setup when a README is meant for public review.
- Include verification commands, local smoke paths, or proof artifacts where practical.
- Make status and limits explicit instead of implying maturity that is not there.
- Prefer local-first tools that leave inspectable files, logs, receipts, or reports behind.
